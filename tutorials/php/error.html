Manejo de errores y ecepciones en php

Estos dos conceptos estan estrechamente relacionados, tinen pocas diferencias. Estos estan representados por 2s clases, Error y Exception, ambas hacen casi lo mismo.

Lanzar un error/eception

En ambos casos la metodologia va a ser muy parecida con un resultado ligeramente diferente.


    function error($a) {
        if ($a === false) {
            throw new Error("Es falso wey! que paja?");
        }
        return "Bien es verdadero";
    }
    function eception($a) {
        if ($a === false) {
            throw new Exception("Es falso wey! que paja?");
        }
        return "Bien es verdadero";
    }
    echo error(false);
    echo eception(false);

El resultado va a ser ligeramente diferente, en ambos casos, lanzara un error, y detendra por completo la ejecucion del codigo, y lanzara.

exception

Fatal error: Uncaught Exception:Es falso wey! que paja?

error

Fatal error: Uncaught Error:Es falso wey! que paja?

seguido de la linea donde se produjo el error,etc.

try catch

Este operador ayuda a ejecutar el codigo hasta que consiga un error, en ese caso detendra la ejecucion del codigo y mandara el mensaje definido y dentendra la ejecucion. pero en el caso del try catch, no mostrara un fatal error ni el mensaje descriptivo de error.

La clase Throwable es la clase padre de Error y Exception. y por lo mismo al usarlo en un try catch, va a ser recivir tanto las excepciones como los errores


    function error($a) {
